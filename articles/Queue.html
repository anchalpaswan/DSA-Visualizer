Queue

Overview

A Queue is a linear data structure that follows the First-In-First-Out (FIFO) principle, where the first element inserted into the queue is the first one to be removed. It can be imagined as a collection of elements arranged in a linear order, and elements are added at the rear (also called "enqueue") and removed from the front (also called "dequeue") of the queue. Queues can be implemented using arrays or linked lists, and they are widely used in computer programming and various applications.

Operations

A queue typically supports the following operations:

  • Enqueue: Insert an element at the rear of the queue.
  • Dequeue: Remove the element from the front of the queue.
  • Peek (or Front): Retrieve the element from the front of the queue without removing it.
  • IsEmpty: Check if the queue is empty or not.
  • IsFull: Check if the queue is full (applicable for queue implementations using arrays).

Applications

Queues have a wide range of applications in computer programming and various domains, including:

  • Managing requests and processing tasks in operating systems
  • Handling printer queues and task scheduling
  • Implementing algorithms like Breadth-First Search (BFS)
  • Handling events and notifications in event-driven systems
  • Buffering and data storage in computer systems
  • Handling message queues and communication protocols
  • Implementing caches and resource pools

    q.push(1);
    q.push(2);
    q.push(3);

    // Dequeuing elements from the queue
    while (!q.empty()) {
        std::cout << "Dequeued: " << q.front() << std::endl;
        q.pop();
    }
